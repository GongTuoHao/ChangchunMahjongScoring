# AGENTS

## 1. 项目目标
- 项目名称：`ChangchunMahjongScoring`
- 目标：实现长春麻将计分逻辑与相关工具能力（规则计算、结果解释、可测试性）。

## 2. 协作原则
- 先读后改：修改前先阅读相关代码与上下文。
- 小步提交：每次变更聚焦一个目标，避免无关改动。
- 可验证：新增或修改逻辑需配套测试或可复现验证步骤。
- 不破坏现有行为：若行为变更，需在说明中明确列出。

## 3. 代码与目录约定
- 保持目录清晰：按 `core/`、`rules/`、`tests/` 等职责拆分（按实际技术栈调整）。
- 文件命名语义化，避免无意义缩写。
- 公共规则与常量集中管理，避免重复定义。

## 4. 计分逻辑约定
- 规则优先数据化：尽量以配置/映射表达番型、分值、优先级。
- 冲突可解释：当多个规则互斥时，输出应包含命中与排除原因。
- 边界明确：对无效输入、缺失字段、非法牌型给出一致错误处理。

## 5. 测试要求
- 至少覆盖：
  - 基础胡牌路径
  - 常见番型组合
  - 互斥规则冲突
  - 边界与异常输入
- 修复缺陷时，优先补回归测试。

## 6. 提交与评审
- 提交信息建议：`type(scope): summary`
- PR/变更说明至少包含：
  - 变更内容
  - 风险点
  - 验证方式

## 7. 技术栈与命令
- 前端技术栈：`Vue 3 + Vite + TypeScript + Tailwind CSS`
- 本地开发：`npm install` 后执行 `npm run dev`
- 类型检查与打包：`npm run build`
- 本地预览构建产物：`npm run preview`
- 静态部署目录：`dist/`（可直接部署到任意静态托管平台）
